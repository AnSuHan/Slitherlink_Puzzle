# 제약 전파 (look-ahead) 로직

`lib/provider/SquareProvider.dart` 의 `propagateLookAhead()` — `checkCurrentPath` 의 `checkMaxLine` / `checkCurrentPathSet` 이후 호출되는 고정점(fixed-point) 시뮬레이션.

## 목적
사용자 입력 직후 셀 숫자 규칙과 꼭짓점 차수 규칙을 반복 적용하여, 더 이상 가능하지 않은 라인을 **비활성(-1)** 으로 마킹한다. 사용자가 잘못 클릭하면 이미 -1 → -2 사이클로 빨간색 표시되므로 별도 X 마킹은 하지 않는다.

## 규칙
1. **셀 숫자 규칙**
   - 그어진 라인 수 == 숫자 → 나머지 라인 비활성.
   - (전체 - 비활성) == 숫자 → 나머지 라인 강제 그어짐(가상 2).
2. **꼭짓점 차수 규칙 (degree must be 0 or 2)**
   - 부정 데드엔드: 그어진 라인 0 + 사용 가능 라인 1 → 그 라인 비활성.
   - 강제 그어짐: 그어진 라인 1 + 사용 가능 라인 1 → 강제 그어짐 (가상 2).
   - 분기 금지: 그어진 라인 ≥ 2 → 나머지 사용 가능 라인 모두 비활성.

## 가상 vs 실제 쓰기
- 시뮬레이션은 readSquare 의 readSubmit/writeSubmit 에서 동작.
- **실제 퍼즐에는 -1(비활성) 만** writeBack. 가상 강제 라인(2)은 다음 입력에서 사용자가 직접 그을 때까지 표시되지 않는다.
- 이유: 자동으로 라인을 그어주면 사용자 게임 경험을 망친다.

## 호출 위치
`checkCurrentPath` 끝부분에서 한 번 실행. 새로 비활성된 라인이 발생할 때마다 다시 전파되도록 루프 종료 조건은 "이번 라운드에 변화 없음".

## 관련 버그
- 이전엔 단순 두 규칙(데드엔드, 분기)만 있었기에 "가장 오른쪽 셀 2 의 오른쪽 라인" 같은 강제 케이스가 잡히지 않았음.
- `updateSquareBox` 에서 `checkCompletePuzzle` 누락도 함께 수정 — 사용자 탭 경로에서 완료 감지가 안되던 문제.
